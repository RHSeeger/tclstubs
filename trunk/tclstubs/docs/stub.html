NAME

stub - Temporarily define/redefine Tcl commands

TABLE OF CONTENTS

    TABLE OF CONTENTS
    SYNOPSIS
    DESCRIPTION
    SEE ALSO
    KEYWORDS

SYNOPSIS

::stubs::stub value name value
::stubs::stub alias name ?args?
::stubs::stub proc name arglist body
::stubs::stub ensemble name subcommand script ?subcommand? ?script? ?...?
::stubs::stub defined name
::stubs::stub group name

DESCRIPTION

The stub command is used to replace (or create) a command temporarily.

::stubs::stub value name value
Replaces the command with one that accepts any number of arguments and returns a static value. 

% ::stubs::stub value myProc 15
% myProc
15



::stubs::stub alias name ?args?
Replaces the command with an alias to the specified command. The optional arguments are passed into the aliased command just like a normal alias. 

% ::stubs::stub alias myProc list 1 2
% myProc a b
1 2 a b



::stubs::stub proc name arglist body
Replaces the command with the specified command definition, just as if defined via proc. 

% ::stubs::stub proc myProc {a b c} {
    eval {$a + $b + $c}
}
% myProc 1 2 3
6



::stubs::stub ensemble name subcommand script ?subcommand? ?script? ?...?
Redefines the actions for certain subcommands of the specified command. Can be used in cases where only certain parts of a command need to be overridden, such as clock seconds. Multiple subcommands of the same command can be overridden in one stub. 

% ::stubs::stub ensemble clock seconds { return 0 }
% clock seconds
0
% clock format 1103523988 -format "%D" -gmt 1
01/01/70



::stubs::stub defined name
Replaces the command with its stub, as defined via stubs::define stub. 

% # Replace array with the builtin version
% # that lsorts 'array names' and 'array get'
% # before returning them
% ::stubs::stub defined array
% array set testArray {c 3 d 4 a 1 b 2}
% array names testArray
a b c d



::stubs::stub group name
Calls the script for the named stub group. This is generally used to stub a variety of related procedures at once.
