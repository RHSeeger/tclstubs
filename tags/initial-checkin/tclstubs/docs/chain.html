NAME

chain - Call the previous definition of a command

TABLE OF CONTENTS

    TABLE OF CONTENTS
    SYNOPSIS
    DESCRIPTION
    EXAMPLES
    SEE ALSO
    KEYWORDS

SYNOPSIS

::stubs::unstub ?commandName?

DESCRIPTION

The chain command is used to call the previous definition of a command. This can be used if a stub definition needs to modify the input or output of a command. If used outside of a stub, it throws an error. The chain command already does an uplevel to make sure that it runs at the same level the original command was run at.

EXAMPLES

% array set testArr {c 10 a 11 d 15 b 1}
% array names testArr
d a b c
% stubs::stub ensemble array names {
    # sort the names before returning them
    return [lsort [eval [linsert $args 0 ::stubs::chain names]]]
}
% array names testArr
a b c d
